# Contribution Guidelines

Thank you for your interest in contributing to the Shopping Intelligence Tool! We appreciate your help in making this project better.

## Getting Started
1. **Fork the Repository:** Start by forking the repository to your personal account.
2. **Clone Your Fork:** Clone your fork to your local machine by running:
   ```
   git clone https://github.com/<your-username>/shopping-intelligence-tool.git
   ```
3. **Set Upstream Remote:** This will help keep your fork up to date with the original repository.
   ```
   git remote add upstream https://github.com/mfataman/shopping-intelligence-tool.git
   ```

## Development Workflow
- **Branching:** Create a new branch for your feature or bug fix. Use descriptive names:
  ```
  git checkout -b feature-name
  ```
- **Make Your Changes:** Implement your feature or bug fix, and write tests to verify your changes.
- **Commit Your Changes:** Commit your changes with a meaningful commit message:
  ```
  git commit -m "Add feature or fix bug"
  ```
- **Push Your Changes:** Push your branch to your fork:
  ```
  git push origin feature-name
  ```

## Submitting Changes
1. **Create a Pull Request:** Navigate to the original repository and create a pull request.
2. **Explain Your Changes:** Provide a clear description of the changes you have made and why they should be merged.
3. **Review Process:** Your pull request will be reviewed. Be open to feedback and make necessary changes if requested.

## Code of Conduct
Please adhere to our [Code of Conduct](CODE_OF_CONDUCT.md) in all interactions.

Thank you for contributing!
